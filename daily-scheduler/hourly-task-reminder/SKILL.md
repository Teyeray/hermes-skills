# hourly-task-reminder

每小时（09:00-21:00）做三件事，优先级从高到低：

1. **到点提醒** — 当前小时有任务，提醒用户
2. **过点重排** — 上一小时有任务过点未完成，自动重排
3. **问一个长期任务** — 没事要提醒时，选一个未规划的长期任务问细节

> ⚠️ 本 skill 由 cronjob `hourly-check`（每天 9-21 点每小时）加载执行。
> **你的最终回复就是发送到微信的消息内容**，cron 框架会自动投递。
>
> 优先执行 1→2→3，只要触发了一个就结束，不要都做。

---

## 执行步骤

### Step 1: 获取当前时间

用 Python 获取当前小时（Asia/Shanghai，24h 制）。

### Step 2: 获取今日任务

```
mcp_dida365_list_undone_tasks_by_time_query(query_command="today")
```

### Step 3: 到点提醒（优先级最高）

检查今日任务中是否有 `startDate` 在当前小时范围内（含具体时间的任务）。

**有 → 回复消息提醒**：
```
⏰ 该做 {任务名} 了！预计 {N} 小时
📅 截止：{dueDate}
```
然后 **结束**，不继续往下走。

**没有 → 跳到 Step 4**

### Step 4: 过点重排（优先级次高）

检查上一小时是否有任务 `status=0`（未完成）且 `startDate` 在上一小时。

**有 → 查找今天下一个空闲时段，用 `mcp_dida365_update_task` 更新到新时间，回复：**
```
🔄 {任务名} 未完成 → 改到 {新时间段}
```
然后 **结束**，不继续往下走。

**没有 → 跳到 Step 5**

### Step 5: 问一个长期任务（最低优先级）

只有当不需要提醒也不需要重排时，才做这个。

1. 用 `mcp_dida365_get_project_with_undone_tasks` 获取 📚 学习 & 研究 项目（id: `6a147ca8e4b0b8cf463cf27f`）的所有未完成任务
2. 遍历任务，找到第一个符合条件的：
   - `startDate` 为 null（完全没安排时间）
   - 或者 `isAllDay=true`（只有日期没有具体时间）
   两者都没有具体时间段，说明还没规划。
3. 选第一个符合条件的，回复一条消息问细节：

```
📋 来安排一个任务～

{任务名}

你每次想做多久？想安排在什么时间（比如每周二四上午）？
想变成习惯定期做吗？

回复我就行，慢慢来
```

**所有任务都已规划 → 回复 `[SILENT]`**，静默跳过。

### Step 6: 完成

---

## 注意事项

- **只做一件事**：1→2→3 优先级，触发一个就结束
- 如果 3 个条件都不满足 → `[SILENT]`
- 问任务时一次只问一个，不要问多个
- 用户回复后会通过微信继续对话，那时再处理后续安排
