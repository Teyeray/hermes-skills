# hourly-task-reminder

每半小时（08:30-21:30）检查是否有还没问过的任务。
如果有，问第一个；用户回复后，AI 在微信对话中继续问下一个，直到全部问完或用户叫停。

> ⚠️ 本 skill 由 cronjob `hourly-check`（每半小时）加载执行。
>
> **关键规则：你的最终回复就是发送到微信的消息内容。**
> 如果全部任务都已问过，回复 `[SILENT]`。

---

## 状态标记

任务是否已问过，通过 `content` 字段判断：
- **已问过**：content 包含 `##state:asked:YYYY-MM-DD##`
- **未问过**：content 不包含

用户在微信上回复后，AI 会在对话中更新 content 加上标记，然后继续问下一个。

---

## 取消词语

当用户不想继续聊了，会说以下词语（或其变体），此时 AI 应停止询问：
- 「先这样吧」
- 「不聊了」
- 「下次再说」
- 「够了」
- 「先不管了」
- 「到此为止」
- 「停」

---

## 执行步骤

### Step 1: 获取所有项目的所有未完成任务

```
mcp_dida365_list_projects
```

对每个项目，调用：
```
mcp_dida365_get_project_with_undone_tasks(project_id="xxx")
```

收集所有未完成的任务。

### Step 2: 找第一个没问过的任务

遍历所有任务，检查 `content` 是否包含 `##state:asked:`。
找到第一个不含此标记的 → 选中它。
全部已问过 → 回复 `[SILENT]`。

### Step 3: 发消息问细节

```
📋 来安排一个任务～

{任务标题}

• 你每次打算花多久时间？
• 想安排在什么时间？（比如每周二四上午）
• 想变成习惯定期做吗？

回复我聊聊，问完这个我会继续问你下一个 👇
如果不想聊了就说「先这样吧」
```

### Step 4: 完成

---

## 微信对话中的延续逻辑（供微信 session 参考）

当用户在微信上回复了某个任务的安排后，AI 应：
1. 用 `mcp_dida365_update_task` 更新任务的 `startDate`、`dueDate` 等
2. 在任务 `content` 末尾追加 `\n##state:asked:YYYY-MM-DD##`
3. 检查是否还有其他未标记的任务
4. **有 → 继续问下一个**（同样的格式，但不用重复说明规则）
5. **没有 → 说"全部安排好了！"**
6. **用户说了取消词语 → 说"好的，那先这样，想继续了随时找我～"** 并停止
