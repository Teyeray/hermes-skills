# hourly-task-reminder

每小时（09:00-21:00）检查 Dida365 任务状态，做两件事：

1. **到点提醒** — 当前小时如果有安排的任务，提醒用户该做了
2. **过点重排** — 上一小时如果有任务过点未完成，自动重新安排

> ⚠️ 本 skill 由 cronjob `hourly-check`（每天 9-21 点每小时）加载执行。
> **你的最终回复就是发送到微信的消息内容**，cron 框架会自动投递。
> 如果当前小时无任务可提醒，回复 **`[SILENT]`** 静默跳过（框架不会发任何消息）。

---

## 执行步骤

### Step 1: 获取当前时间

用 Python 或系统时间获取当前的小时数（Asia/Shanghai 时区，24h 制）。

### Step 2: 获取今日任务

```
mcp_dida365_list_undone_tasks_by_time_query(query_command="today")
```

### Step 3: 分析当前小时是否有任务

检查今日任务中是否有任务的 `startDate` 落在当前小时范围内。

判断逻辑：
- 如果任务的 `startDate` 为 `null` → 该任务没有安排具体时间，跳过
- 如果任务的 `startDate` 在当前小时（如 10:00-10:59）范围内 → 该任务应在此小时做

**情况 A：有任务应在本小时做**
→ 回复消息：

```
⏰ 提醒：该做 {任务名} 了！
   ⏱ 预计 {N} 小时
   📅 截止：{dueDate}
   🔴/🟡 优先级：{高/中}
```

**情况 B：当前小时无任务安排**
→ **回复 `[SILENT]`**（静默跳过，框架不会发消息）。

### Step 4: 检查上一小时是否有任务过点未完成

查看上一小时（如当前是 11:00，则看 10:00-10:59）安排的任务。

判断逻辑：
- 任务状态 `status=0`（未完成）
- 任务 `startDate` 在上一小时范围内

**如果有任务过点未完成：**
1. 查找今天下一个空闲时段（从当前小时往后找，避开已安排的时间段）
2. 用 `mcp_dida365_update_task` 更新该任务的 `startDate` 和 `dueDate` 到新的时段
3. 在回复中追加：

```
🔄 重新安排：{任务名} 未在预定时间完成
   原时间：{原时间段}
   新时间：{新时间段}
```

**如果没有任务过点未完成：**
→ 不需要追加内容。

### Step 5: 完成

生成最终回复后即完成。

---

## 注意事项

- **静默是关键** — 本小时无任务时**必须**回复 `[SILENT]`，不要发空消息
- 只检查上一小时 + 当前小时，不要扫描全天
- 重新安排时，优先填到当天的空闲时段；如果当天已没有空闲，推到明天（更新日期 +1）
- 更新 Dida365 任务时，只改 `startDate` 和 `dueDate`，不要改其他字段
- 如果任务是在 Dida365 中被用户手动完成的（`status=2`），不要发消息询问
