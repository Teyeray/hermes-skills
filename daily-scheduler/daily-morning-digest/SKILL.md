# daily-morning-digest

每天早晨分析 Dida365 中的任务，**在 Dida365 中实际设置每个任务的具体时间段**，
然后生成日程安排推送到微信。

> ⚠️ 本 skill 由 cronjob `morning-digest`（每天 08:00）加载执行。
>
> **关键规则：你的最终回复就是发送到微信的消息本身。**
> ❌ 不要在回复中包含你的思考过程、分析步骤、代码、JSON 等
> ✅ 回复内容 = 用户将在微信上看到的纯消息
>
> 如果今日无任务，回复 `[SILENT]` 跳过。

---

## 执行步骤

### Step 1: 获取今日任务

```
mcp_dida365_list_undone_tasks_by_time_query(query_command="today")
```

### Step 2: 获取本周其他任务

计算本周一 ~ 本周日的日期（Asia/Shanghai），调用：
```
mcp_dida365_filter_tasks(filter={
  "startDate": "<本周一 00:00:00+08:00>",
  "endDate": "<本周日 23:59:59+08:00>"
})
```

### Step 3: 判断短时 vs 长期任务 ⭐

对每个今日任务，严格判断：

**短时任务（一次能搞定，不用问）**：只有以下明确场景才算短时：
- 取消订阅/会员
- 查一条信息/看一个网页
- 发一封邮件/消息
- 改一个密码/设置
- 下载一个文件
- 填一个表单
- 打一个电话

**其余全部算长期任务**：需要规划时间、分多次做的
- 学习、写论文、刷题、做项目、整理资料、上课程
- 任何包含"梳理""整理""学习""研究""准备""论文"等关键词的
- 任何你不确定能不能一次搞定的

### Step 4: 短时任务 → 直接安排时间

对短时任务，按优先级在 Dida365 中设置具体时间段：

```
09:00-10:00  优先级 5 的短时任务（黄金时段）
14:00-15:00  优先级 3 的短时任务
17:00-17:30  优先级 1 或 0 的短时任务
```

用 `mcp_dida365_update_task` 设置 `startDate` 和 `dueDate`。

### Step 5: 长期任务 → 不安排时间，只标记日期

对长期任务，**不要设置具体的开始/结束时间**（因为需要你告诉用户怎么安排）。
只确保它们的 `dueDate` 是正确的截止日期。

### Step 6: 组装最终回复（即微信消息）

```
☀️ 早安！{月/日 周X}

━━━ 今日安排 ━━━━━━━━━━━━━

🔴 高优
  09:00-10:00  {短时任务A}
  ...

🟡 中优
  14:00-15:00  {短时任务B}
  ...

━━━ 本周还有 ━━━━━━━━━━━━━
□ {任务名} ({截止日})
□ {任务名} ({截止日})

━━━ 📋 需要聊聊安排 ━━━━━━━
以下任务需要你告诉我怎么规划，直接回复我就行👇

• 个人论文 → 每次想做多久？想安排在什么时间？
• 强化学习 → 每周想做几次？每次多久？
• 整理网盘 → 想一次性搞完还是分批做？

━━━ ⚠️ 提醒 ━━━━━━━━━━━━━━
{风险点列表}

━━━
📌 有新任务直接在微信上告诉我就行
✅ 任务做完在 Dida365 里勾一下
```

---

## 注意事项

- **最终回复 = 微信消息**。不要输出分析过程、代码、JSON、步骤说明
- 以 `☀️ 早安！` 开头
- 对长期任务，在"需要聊聊安排"部分提问，每个任务问 1 个关键问题即可
- 不要一次性问太多，3-4 个为上限
- 不设置长期任务的具体时间（让用户决定）
- 如果今日无任务，回复 `[SILENT]`
