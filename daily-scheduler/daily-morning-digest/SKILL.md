# daily-morning-digest

每天早晨分析 Dida365 中的任务，安排短时任务的时间，加上天气预报，生成晨报推送到微信。

> ⚠️ 本 skill 由 cronjob `morning-digest`（每天 08:00）加载执行。
>
> **关键规则：你的最终回复就是发送到微信的消息本身。**
> ❌ 不要在回复中包含思考过程、分析步骤、代码、JSON 等
> ✅ 回复内容 = 用户将在微信上看到的纯消息
>
> 如果今日无任务且本周也无任务，回复 `[SILENT]` 跳过。

---

## 执行步骤

### Step 1: 获取今日任务

```
mcp_dida365_list_undone_tasks_by_time_query(query_command="today")
```

### Step 2: 获取本周其他任务

计算本周一 ~ 本周日的日期（Asia/Shanghai），调用：
```
mcp_dida365_filter_tasks(filter={
  "startDate": "<本周一 00:00:00+08:00>",
  "endDate": "<本周日 23:59:59+08:00>"
})
```

### Step 3: 获取天气

用 terminal 获取上海天气（中文化）：
```
curl -s "wttr.in/Shanghai?format=%C+%t&lang=zh"
```

如果 curl 失败或返回空，跳过天气部分。

### Step 4: 判断短时 vs 长期任务

**短时任务（一次能搞定）**：只有以下明确场景才算：
- 取消订阅/会员
- 查一条信息/看一个网页
- 发一封邮件/消息
- 改一个密码/设置
- 下载一个文件
- 填一个表单
- 打一个电话

**其余全部算长期任务**。

### Step 5: 短时任务 → 直接安排时间

对短时任务，按优先级在 Dida365 中设置具体时间段（09:00-12:00 高优，14:00-17:00 中低优）。
用 `mcp_dida365_update_task` 设置 `startDate` 和 `dueDate`（isAllDay=false）。

### Step 6: 组装最终回复（即微信消息）

```
☀️ 早安！{月/日 周X}  🌤 {天气}

━━━ 今日安排 ━━━━━━━━━━━━━
09:00-10:00  {短时任务A} 🔴
14:00-15:00  {短时任务B} 🟡

━━━ 本周还有 ━━━━━━━━━━━━━
□ {任务名} ({月/日 周X} 截止)
□ {任务名} ({月/日 周X} 截止)

━━━ 长期任务（待规划） ━━━
📚 学习 & 研究：{N} 个待安排
⚙️ 系统 & 维护：{N} 个待处理

我会每小时逐个问你安排细节～

━━━
📌 有新任务直接微信告诉我
✅ 做完在 Dida365 里勾一下
```

### Step 7: 完成

## 注意事项

- 最终回复 = 纯微信消息，不要包含任何思考过程
- 以 `☀️ 早安！` 开头，接天气
- 今日无任务但有本周任务 → 发本周概览
- 今日和本周都无任务 → `[SILENT]`
