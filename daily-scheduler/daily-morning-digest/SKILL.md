# daily-morning-digest

每天早晨分析 Dida365 中的任务，生成今日日程安排并推送到微信。

> ⚠️ 本 skill 由 cronjob `morning-digest`（每天 08:00）加载执行。
> 执行完毕后用 `send_message` 将结果发送到微信平台，
> target = `weixin:o9cq808c1Lky6PCGruZT10z1Qz8Y@im.wechat`

---

## 执行步骤

### Step 1: 获取今日任务

调用：
```
mcp_dida365_list_undone_tasks_by_time_query(query_command="today")
```

返回今日所有未完成的任务列表。每个任务包含：`id`, `title`, `priority`, `startDate`, `dueDate`, `tags`, `kind` 等字段。

### Step 2: 获取本周任务

计算本周一 ~ 本周日的日期范围（以 Asia/Shanghai 时区为准），调用：
```
mcp_dida365_filter_tasks(filter={
  "startDate": "<本周一 00:00:00+08:00>",
  "endDate": "<本周日 23:59:59+08:00>"
})
```

### Step 3: 时间估算

按优先级给每个任务估算所需时间（默认规则，无需用户配置）：

| 优先级 | 含义 | 估算时间 |
|--------|------|---------|
| `priority=5` | 最高 | 3 小时 |
| `priority=3` | 中 | 2 小时 |
| `priority=1` | 低 | 1 小时 |
| `priority=0` | 无 | 0.5 小时 |

### Step 4: 排日程

将今日任务按以下规则填入时间轴：

- **黄金时段 09:00-12:00** → 安排优先级 5 的任务（如果放不下，溢出到下午）
- **下午时段 14:00-17:00** → 安排优先级 3 的任务
- **灵活时段 17:00-18:00** → 安排优先级 1 或 0 的任务
- 同一任务连续安排不超过 3 小时，中间留 10 分钟缓冲
- 如果某任务已有 `startDate` 且在今天，优先保留用户设定的时间

### Step 5: 风险检测

检查以下风险，有则标记：

| 风险类型 | 判断条件 |
|---------|---------|
| ⚠️ 超载 | 今日预计总时间 > 6 小时 |
| ⚠️ 拖延 | 同一高优任务在 Dida365 中超过 3 天未完成 |
| ⚠️ 冲突 | 多个高优任务的截止日期重叠在同一天 |

### Step 6: 组装消息并发送

用以下格式组装消息，通过 `send_message` 发送到微信：

```
☀️ 早安！{月/日 周X}

━━━ 今日待办 ━━━━━━━━━━━━━

🔴 高优
  □ {任务名} ({时间段})
  □ {任务名} ({时间段})

🟡 中优
  □ {任务名} ({时间段})

🔵 低优
  □ {任务名} ({时间段})

━━━ 时间轴 ━━━━━━━━━━━━━━━
09:00-12:00  {任务名} 🔴
14:00-16:00  {任务名} 🟡
16:00-17:00  {任务名} 🔵

━━━ 本周还有 ━━━━━━━━━━━━━
□ {任务名} ({截止日})
□ {任务名} ({截止日})

━━━ ⚠️ 风险提醒 ━━━━━━━━━━━
{风险点列表，无风险则显示 ✔ 今日安排合理}

━━━
📌 有新任务请在 Dida365 中添加
✅ 完成任务后请在 Dida365 中勾选
```

### Step 7: 完成

发送后即完成，不需要额外操作。

---

## 注意事项

- 如果今日无任务，发送：`☀️ 早安！今天没有待办事项，好好休息或规划一下新任务吧 📝`
- 消息发到微信后，cronjob 会自动结束
- 不要在 Dida365 中修改任何任务数据（只读操作），时间安排只体现在消息中，实际调整由 evening-review 处理
