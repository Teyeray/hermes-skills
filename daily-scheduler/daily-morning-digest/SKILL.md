# daily-morning-digest

每天早晨分析 Dida365 中的任务，**在 Dida365 中实际设置每个任务的具体时间段**，
然后生成日程安排推送到微信。

> ⚠️ 本 skill 由 cronjob `morning-digest`（每天 08:00）加载执行。
> **你的最终回复就是发送到微信的消息内容**，cron 框架会自动投递。
> 如果今日无任务需要安排，回复 `[SILENT]` 跳过发送。

---

## 执行步骤

### Step 1: 获取今日任务

```
mcp_dida365_list_undone_tasks_by_time_query(query_command="today")
```

### Step 2: 获取本周其他任务

计算本周一 ~ 本周日的日期范围（Asia/Shanghai），调用：
```
mcp_dida365_filter_tasks(filter={
  "startDate": "<本周一 00:00:00+08:00>",
  "endDate": "<本周日 23:59:59+08:00>"
})
```
过滤掉已完成的，只保留未完成的任务。

### Step 3: 检查任务日期的合理性

对每个任务检查标题与日期的匹配：
- 如果标题含"周五"但日期不是周五 → 修正到最近的周五
- 如果标题含"周一/周二/..."但日期不匹配 → 修正
- 如果截止日期已经过了 → 修正到今天或最近的合理日期
- 用 `mcp_dida365_update_task` 更新修正后的日期

### Step 4: 时间估算

按优先级给每个任务估算所需时间：

| 优先级 | 含义 | 估算时间 | 典型任务 |
|--------|------|---------|---------|
| `priority=5` | 最高 | 3 小时 | 紧急任务、面试准备 |
| `priority=3` | 中 | 2 小时 | 学习、论文、项目 |
| `priority=1` | 低 | 1 小时 | 兴趣、探索性任务 |
| `priority=0` | 无 | 0.5 小时 | 琐事、一次性操作 |

对于 `CHECKLIST` 类型的任务（如"整理网盘"有3个子项），按子项数量 × 1小时估算。

### Step 5: 在 Dida365 中设置时间

对每个今日任务，用 `mcp_dida365_update_task` 设置具体的开始和结束时间：

排程规则：
- **09:00-12:00** → 优先级 5 的任务（黄金时段）
- **14:00-17:00** → 优先级 3 的任务
- **17:00-18:00** → 优先级 1 或 0 的任务
- 每个任务之间留 10 分钟缓冲
- 如果任务已有 `startDate` 且在今天，保留用户设定的时间
- 如果今天任务太多放不下，把低优先级推到明天

更新示例：
```python
mcp_dida365_update_task(
  task_id="xxx",
  task={
    "id": "xxx",
    "projectId": "inbox1026706825",
    "startDate": "2026-05-26T09:00:00+08:00",
    "dueDate": "2026-05-26T12:00:00+08:00",
    "isAllDay": false
  }
)
```

### Step 6: 风险检测

| 风险类型 | 判断条件 |
|---------|---------|
| ⚠️ 超载 | 今日预计总时间 > 6 小时 |
| ⚠️ 拖延 | 同一高优任务在 Dida365 中超过 3 天未完成 |
| ⚠️ 冲突 | 多个高优任务的截止日期重叠在同一天 |

### Step 7: 组装最终回复（即微信消息）

```
☀️ 早安！{月/日 周X}

━━━ 今日安排 ━━━━━━━━━━━━━

🔴 高优
  09:00-12:00  {任务名}
  ...

🟡 中优
  14:00-16:00  {任务名}
  ...

🔵 低优
  17:00-18:00  {任务名}
  ...

━━━ 本周还有 ━━━━━━━━━━━━━
□ {任务名} ({截止日})
□ {任务名} ({截止日})

━━━ ⚠️ 风险 ━━━━━━━━━━━━━
{风险点列表，无风险则显示 ✔ 今日安排合理}

━━━
📌 新任务请在 Dida365 中添加
✅ 完成记得在 Dida365 中勾选
```

### Step 8: 完成

---

## 注意事项

- **必须在 Dida365 中写时间**（Step 5），不只是消息里展示
- 如果今日无任务，回复 `[SILENT]`
- 更新任务时记得传 `projectId`
- 日期合理性检查（Step 3）只修正明显不合理的日期，不要过度修改
