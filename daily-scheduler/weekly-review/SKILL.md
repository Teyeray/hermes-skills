# weekly-review

每周日晚上 20:00 总结本周任务完成情况，预测下周任务量，发送周报到微信。

> ⚠️ 本 skill 由 cronjob `weekly-review`（每周日 20:00）加载执行。

---

## 执行步骤

### Step 1: 获取本周任务

计算本周一 ~ 本周日的日期范围（Asia/Shanghai 时区）：

```
mcp_dida365_filter_tasks(filter={
  "startDate": "<本周一 00:00:00+08:00>",
  "endDate": "<本周日 23:59:59+08:00>"
})
```

### Step 2: 获取本周已完成任务

```
mcp_dida365_list_completed_tasks_by_date(search={
  "startDate": "<本周一 00:00:00+08:00>",
  "endDate": "<本周日 23:59:59+08:00>"
})
```

### Step 3: 统计完成情况

- 总任务数
- 已完成数
- 未完成数
- 完成率
- 按优先级统计：

| 优先级 | 总数 | 已完成 | 未完成 |
|--------|------|--------|--------|
| 🔴 高 (P5) | N | X | Y |
| 🟡 中 (P3) | N | X | Y |
| 🔵 低 (P1) | N | X | Y |
| ⚪ 无 (P0) | N | X | Y |

### Step 4: 预测下周

计算下周一的日期，获取所有截止日期在下周的任务：

```
mcp_dida365_filter_tasks(filter={
  "startDate": "<下周一 00:00:00+08:00>",
  "endDate": "<下周日 23:59:59+08:00>"
})
```

统计下周预计任务量，标记风险：
- 下周任务量 > 本周 × 1.5 → ⚠️ 下周任务量较大，提前规划
- 下周有高优任务 → 🔴 注意提前准备

### Step 5: 发送周报

```
📊 周报 | 第X周 ({月/日 - 月/日})

━━━ 本周完成 ━━━━━━━━━━━
✅ 已完成：{N} / {M} 个
📊 完成率：{XX}%

━━━ 按优先级 ━━━━━━━━━━━
🔴 高优：{X}/{Y} 已完成
🟡 中优：{X}/{Y} 已完成
🔵 低优：{X}/{Y} 已完成

━━━ 未完成 ━━━━━━━━━━━━━
□ {任务名}（已顺延）
□ {任务名}（已顺延）

━━━ 下周预告 ━━━━━━━━━━━
📋 预计 {N} 个任务待处理
{风险提示}

━━━
📌 新任务请在 Dida365 中添加
```

### Step 6: 完成

---

## 注意事项

- 本 skill 只读不写（不修改任何 Dida365 数据），只做统计和推送
- 如果本周无任务，发送：`📊 本周没有任务记录，是休息周吗？好好放松一下 🎉`
- 如果完成率 > 80%，在消息末尾加一句褒奖
- 如果完成率 < 40%，加一句鼓励
