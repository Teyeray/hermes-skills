# daily-evening-review

每天晚上 20:00 分析今日任务完成情况，将未完成的任务在 Dida365 中重新安排到明天，并发送今日总结到微信。

> ⚠️ 本 skill 由 cronjob `evening-review`（每天 20:00）加载执行。

---

## 执行步骤

### Step 1: 获取今日所有任务

先获取今日未完成的任务：
```
mcp_dida365_list_undone_tasks_by_time_query(query_command="today")
```

再获取今日已完成的任务：
```
mcp_dida365_list_completed_tasks_by_date(search={
  "startDate": "<今天 00:00:00+08:00>",
  "endDate": "<今天 23:59:59+08:00>"
})
```

### Step 2: 计算完成情况

- 今日任务总数 = 已完成数 + 未完成数
- 完成率 = 已完成数 / 总任务数 × 100%

### Step 3: 处理未完成的任务

对每个 **status=0**（未完成）的任务：

1. 计算明天日期
2. 调用 `mcp_dida365_update_task` 更新任务：
   ```
   task={
     "id": "<任务id>",
     "projectId": "<任务所在项目id>",
     "startDate": "<明天 00:00:00+08:00>",
     "dueDate": "<明天 23:59:59+08:00>"
   }
   ```
3. 记录该任务已被重排

如果任务 `startDate` 和 `dueDate` 均为 null（无日期任务），则**不修改**，保持原样。

### Step 4: 组装今日总结并发送

```
🌆 今日总结 | {月/日 周X}

━━━ 完成情况 ━━━━━━━━━━━━
✅ 已完成：{N} 个
⏳ 未完成：{M} 个
📊 完成率：{XX}%

━━━ 已完成 ━━━━━━━━━━━━━
□ {任务名} ✓
□ {任务名} ✓

━━━ 已重排到明天 ━━━━━━━
□ {任务名} → 明天
□ {任务名} → 明天

━━━ 未安排日期的任务 ━━━
□ {任务名}（无日期，保持原样）

━━━
💪 明天继续加油！
```

### Step 5: 完成

---

## 注意事项

- 更新 Dida365 任务时，必须同时传入 `projectId`，否则 API 会失败
- 无日期的任务不要改（可能是不紧急的长期事项）
- 如果今天所有任务都完成了，发送庆祝消息：
  ```
  🎉 今日全部完成！{N} 个任务已搞定，太棒了！
  ```
- 如果今天本来就没有任务，静默跳过，不发消息
