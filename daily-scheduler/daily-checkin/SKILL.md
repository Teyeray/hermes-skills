# daily-checkin

每天在固定时间（如 12:00 或 18:00）主动与用户聊天互动，了解用户在想什么，
根据当前任务情况给出建议，帮用户安排新任务到 Dida365。

> ⚠️ 本 skill 由 cronjob `daily-checkin` 加载执行。
> **你的最终回复就是发送到微信的消息内容**，cron 框架会自动投递。
> 消息要像朋友一样自然聊天，不要太生硬。

---

## 执行步骤

### Step 1: 获取今日任务完成情况

```
mcp_dida365_list_undone_tasks_by_time_query(query_command="today")
```

以及今日已完成任务：
```
mcp_dida365_list_completed_tasks_by_date(search={
  "startDate": "<今天 00:00:00+08:00>",
  "endDate": "<今天 23:59:59+08:00>"
})
```

### Step 2: 获取本周整体情况

```
mcp_dida365_filter_tasks(filter={
  "startDate": "<本周一 00:00:00+08:00>",
  "endDate": "<本周日 23:59:59+08:00>"
})
```

关注：
- 本周高优任务还剩哪些
- 哪些即将到截止日期
- 用户的面试准备进度（🔍面试prepare 项目）

### Step 3: 判断当前时间

根据当前时间决定聊天语气：
- **12:00（中午）** → 轻松问候，问上午进展，建议下午安排
- **18:00（傍晚）** → 问今天整体感受，给晚上建议
- **15:00（下午）** → 简短提醒，问问状态

### Step 4: 组装聊天消息

消息要像朋友聊天一样自然，包含以下元素：

1. **问候** — 根据时间友好的招呼
2. **进度感知** — 知道用户今天做了什么、没做什么
3. **开放问题** — 问用户在想什么/有什么计划
4. **建议** — 根据任务情况给出1-2个具体建议
5. **主动帮忙** — 询问是否需要帮用户在 Dida365 中添加新任务

示例消息（中午12:00）：
```
👋 中午好！上午过得怎么样？

我看到你今天还有：
• {任务A}（还没开始）
• {任务B}（进行中）

下午有什么打算？要不要先把 {某个高优任务} 搞定？

最近在看 {任务主题} 相关的东西吗？有什么想法可以聊聊～

如果有新的任务要加，直接告诉我就好，我帮你记到 Dida365 📝
```

示例消息（傍晚18:00）：
```
🌆 傍晚好！今天感觉如何？

✅ 完成了 {N} 个任务
⏳ 还有 {M} 个没搞定

今天有什么收获或想法想分享吗？

关于 {某个任务}，要不要我帮你查点资料或给些建议？

另外，明天有什么新计划吗？随时告诉我就行～
```

### Step 5: 完成

回复内容就是微信消息。用户收到后如果想聊，直接在微信回复即可。

---

## 注意事项

- **语气要自然！** 不要像机器人，要像朋友聊天
- 不要一次性给太多建议（最多2条）
- 如果用户今天很忙（很多高优任务），语气要简洁务实
- 如果用户今天很轻松（任务少），可以聊得轻松随意些
- 消息末尾留一个开放式的邀请，让用户愿意回复
- 用户回复后会在微信上和 Hermes 继续对话，这是正常的
